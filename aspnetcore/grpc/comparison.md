## <a name="grpc-strengths"></a><span data-ttu-id="ef055-138">gRPC の長所</span><span class="sxs-lookup"><span data-stu-id="ef055-138">gRPC strengths</span></span>

### <a name="performance"></a><span data-ttu-id="ef055-139">パフォーマンス</span><span class="sxs-lookup"><span data-stu-id="ef055-139">Performance</span></span>

<span data-ttu-id="ef055-140">gRPC メッセージは、効率的なバイナリ メッセージ形式である [Protobuf](https://developers.google.com/protocol-buffers/docs/overview) を使用してシリアル化されます。</span><span class="sxs-lookup"><span data-stu-id="ef055-140">gRPC messages are serialized using [Protobuf](https://developers.google.com/protocol-buffers/docs/overview), an efficient binary message format.</span></span> <span data-ttu-id="ef055-141">Protobuf はサーバーおよびクライアント上で、非常に高速にシリアル化を行います。</span><span class="sxs-lookup"><span data-stu-id="ef055-141">Protobuf serializes very quickly on the server and client.</span></span> <span data-ttu-id="ef055-142">Protobuf のシリアル化では、メッセージ ペイロードが小さくなります。これはモバイル アプリのように帯域幅が限られるシナリオでは重要です。</span><span class="sxs-lookup"><span data-stu-id="ef055-142">Protobuf serialization results in small message payloads, important in limited bandwidth scenarios like mobile apps.</span></span>

<span data-ttu-id="ef055-143">gRPC は、HTTP のメジャー リビジョンである HTTP/2 向けに設計されており、パフォーマンスは HTTP 1.x よりも大幅に向上します。</span><span class="sxs-lookup"><span data-stu-id="ef055-143">gRPC is designed for HTTP/2, a major revision of HTTP that provides significant performance benefits over HTTP 1.x:</span></span>

* <span data-ttu-id="ef055-146">単一 TCP 接続での複数の HTTP/2 呼び出しの多重化。</span><span class="sxs-lookup"><span data-stu-id="ef055-146">Multiplexing of multiple HTTP/2 calls over a single TCP connection.</span></span> <span data-ttu-id="ef055-147">多重化により、[ヘッドオブライン ブロッキング](https://en.wikipedia.org/wiki/Head-of-line_blocking)が解消されます。</span><span class="sxs-lookup"><span data-stu-id="ef055-147">Multiplexing eliminates [head-of-line blocking](https://en.wikipedia.org/wiki/Head-of-line_blocking).</span></span>

<span data-ttu-id="ef055-148">HTTP/2 は gRPC に限定されません。</span><span class="sxs-lookup"><span data-stu-id="ef055-148">HTTP/2 is not exclusive to gRPC.</span></span> <span data-ttu-id="ef055-149">JSON を使用する HTTP API など、多くの要求の種類で、HTTP/2 を使用すると、パフォーマンスの向上による恩恵を受けることができます。</span><span class="sxs-lookup"><span data-stu-id="ef055-149">Many request types, including HTTP APIs with JSON, can use HTTP/2 and benefit from its performance improvements.</span></span>

## <a name="grpc-weaknesses"></a><span data-ttu-id="ef055-186">gRPC の短所</span><span class="sxs-lookup"><span data-stu-id="ef055-186">gRPC weaknesses</span></span>

<span data-ttu-id="ef055-188">現在のところ、ブラウザーから gRPC サービスを直接呼び出すことはできません。</span><span class="sxs-lookup"><span data-stu-id="ef055-188">It's impossible to directly call a gRPC service from a browser today.</span></span> <span data-ttu-id="ef055-189">gRPC は HTTP/2 の機能を多用しており、gRPC クライアントをサポートするための Web 要求を通して必要とされるレベルの制御を提供するブラウザーがありません。</span><span class="sxs-lookup"><span data-stu-id="ef055-189">gRPC heavily uses HTTP/2 features and no browser provides the level of control required over web requests to support a gRPC client.</span></span> <span data-ttu-id="ef055-190">たとえば、ブラウザーでは呼び出し元に、HTTP/2 を使用するよう求めたり、基になる HTTP/2 フレームへのアクセスを提供したりすることを許可していません。</span><span class="sxs-lookup"><span data-stu-id="ef055-190">For example, browsers do not allow a caller to require that HTTP/2 be used, or provide access to underlying HTTP/2 frames.</span></span>

### <a name="not-human-readable"></a><span data-ttu-id="ef055-202">人が判読できない</span><span class="sxs-lookup"><span data-stu-id="ef055-202">Not human readable</span></span>

<span data-ttu-id="ef055-203">HTTP API 要求はテキストとして送信され、人が読み取りと作成を行えます。</span><span class="sxs-lookup"><span data-stu-id="ef055-203">HTTP API requests are sent as text and can be read and created by humans.</span></span>

<span data-ttu-id="ef055-204">gRPC メッセージは、既定では Protobuf でエンコードされます。</span><span class="sxs-lookup"><span data-stu-id="ef055-204">gRPC messages are encoded with Protobuf by default.</span></span> <span data-ttu-id="ef055-205">Protobuf での送受信は効率的ですが、そのバイナリ形式は人が判読できません。</span><span class="sxs-lookup"><span data-stu-id="ef055-205">While Protobuf is efficient to send and receive, its binary format isn't human readable.</span></span> <span data-ttu-id="ef055-206">Protobuf では、正しく逆シリアル化するために、 *.proto* ファイルで指定されているメッセージのインターフェイスに関する説明を必要とします。</span><span class="sxs-lookup"><span data-stu-id="ef055-206">Protobuf requires the message's interface description specified in the *.proto* file to properly deserialize.</span></span> <span data-ttu-id="ef055-207">ネットワーク上の Protobuf ペイロードを分析し、手動で要求を作成するには、追加のツールが必要です。</span><span class="sxs-lookup"><span data-stu-id="ef055-207">Additional tooling is required to analyze Protobuf payloads on the wire and to compose requests by hand.</span></span>

<span data-ttu-id="ef055-208">バイナリの Protobuf メッセージをサポートするため、[サーバー リフレクション](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md)や [gRPC コマンド ライン ツール](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md)などの機能が存在します。</span><span class="sxs-lookup"><span data-stu-id="ef055-208">Features such as [server reflection](https://github.com/grpc/grpc/blob/master/doc/server-reflection.md) and the [gRPC command line tool](https://github.com/grpc/grpc/blob/master/doc/command_line_tool.md) exist to assist with binary Protobuf messages.</span></span> <span data-ttu-id="ef055-209">また、Protobuf メッセージでは、[JSON との間の変換](https://developers.google.com/protocol-buffers/docs/proto3#json)がサポートされています。</span><span class="sxs-lookup"><span data-stu-id="ef055-209">Also, Protobuf messages support [conversion to and from JSON](https://developers.google.com/protocol-buffers/docs/proto3#json).</span></span> <span data-ttu-id="ef055-210">組み込みの JSON 変換によって、デバッグ時に Protobuf メッセージを人が判読できる形式に変換する効率的な方法が提供されます。</span><span class="sxs-lookup"><span data-stu-id="ef055-210">The built-in JSON conversion provides an efficient way to convert Protobuf messages to and from human readable form when debugging.</span></span>

## <a name="alternative-framework-scenarios"></a><span data-ttu-id="ef055-211">他のフレームワークのシナリオ</span><span class="sxs-lookup"><span data-stu-id="ef055-211">Alternative framework scenarios</span></span>

<span data-ttu-id="ef055-212">以下のシナリオでは、gRPC よりも他のフレームワークが推奨されます。</span><span class="sxs-lookup"><span data-stu-id="ef055-212">Other frameworks are recommended over gRPC in the following scenarios:</span></span>
